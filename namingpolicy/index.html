Naming Policy Violations

3758 packages: 3572 Correct; 22 Misnamed Subpackage; 128 Ambiguous Requires; 36 Blocked

This is a dashboard to track various naming scheme violations in Python packages:

  • Misnamed Subpackage - naming scheme of (at least) one of the binary RPMs does not comply with the new Python package naming guidelines
  • Ambiguous Requires - some build-time or run-time requirements are of type python-foo, although the names with a versioned Python prefix (e.g. python2-foo) are available
  • Blocked - some build-time or run-time requirements are of type python-foo, because the dependency is misnamed and the name python2-foo is not available
We encourage to follow the new guidelines to avoid a range of issues when upstream support for Python 2 ends and python means Python 3 in Fedora.
